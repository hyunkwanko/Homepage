git checkout HEAD -filename : 특정 파일을 원하는 Commit 시점으로 되돌리기


 Sourcetree 실행 시 첫 화면


 Local에서 디렉토리 불러온 후 현재 Commit 상태 확인


 파일명은 'README.md'이고 1줄에 1 Commit했다.


 되돌아가길 원하는 Commit을 우클릭 후 '체크아웃' 버튼 클릭


 확인창


 Commit HEAD가 바뀐 것을 확인


 파일의 내용을 확인해보면 바뀐 것을 알 수 있다.


 번외로 이 명령어를 bash에서 실행하면 '원하는' Commit 시점이 아니라 '최종' Commit 시점으로 돌아간다. 하지만 Sourcetree에선 원하는 시점으로 갈 수 있다.


 현재 HEAD가 세 번째 Commit을 가리키고 있다. 첫 번째 Commit으로 돌아가보겠다.


 이전 Commit이 아니라서 다른 확인창이 나오는 것을 확인


 새 브랜치로 체크아웃도 할 수 있지만 기존의 브랜치에서 체크아웃


 Sourcetree에서 각각의 Commit과 그 내용을 한 눈에 알아볼 수 있도록 GUI를 제공
 첫 번째 Commit으로 체크아웃


 경고창


 Commit HEAD가 첫 번째 Commit으로 바뀐 것을 확인


 파일의 내용 또한 바뀌었다.


 다시 마지막 Commit으로 돌아가보겠다.


 이번엔 경고창이 나오지 않고 Commit HEAD가 바뀐 모습이다.


 파일의 내용 또한 마지막 Commit과 동일
